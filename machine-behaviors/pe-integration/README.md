# pe-integration

Materialized PE source mappings for the corpus-wide OpenClaw agent automation —
the "mappings to PE sources for the responses".

- **`corpus.pe-source-mappings.json`** — one source mapping per write-back agent
  (1047), generated by `domain_sweep.py --all --write`. Shape mirrors
  `RealityEngine_CPP/config/integrations.example.json` `sourceMappings`. Each
  entry registers an agent's completion sensor
  (`acp.openclaw.<machine>.<agent>.completion`) at its reserved-band region with
  `extract.pointers` matching the response mapping positions.

## Use

Merge `sourceMappings` into the PE `INTEGRATIONS_CONFIG` so PE ingests OpenClaw
completions via `POST /api/integrations/completions` and RE reads them on the next
push cycle. All regions live in the registry-reserved band
`rangePolicy.reservedRanges["acp-openclaw-agent-completions"]` (`[7300:12300]`).

## Regenerate

```bash
cd .. && MB_DEBUG=0 python3.13 domain_sweep.py --all --write
cp out/corpus.pe-source-mappings.json pe-integration/corpus.pe-source-mappings.json
```

This file is a committed snapshot (the deliverable); the full per-agent dump
`out/corpus.agents.json` (15 MB) stays gitignored and regenerable.
